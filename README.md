# أثَر | ملحمة كربلاء 🌟
> **رؤية رقمية معاصرة لنشر سيرة الإمام الحسين (ع) ورجالات الطف البررة**

منصة **أثَر** الرقمية هي متحف رقمي وتطبيق تفاعلي ويب متكامل ومصقول يوثق الأحداث الكاملة لـ **ملحمة كربلاء** التاريخية وسيرة سيد الشهداء الإمام الحسين بن علي (ع) وأصحابه البررة، بالاعتماد على النصوص التاريخية من كتاب **بحار الأنوار** للعلامة المجلسي. يهدف التطبيق إلى تقديم هذه الملحمة المباركة بطريقة معاصرة تدمج بين جماليات التصميم والمزايا التفاعلية الرقمية لتسهيل القراءة والبحث والدراسة.

---

## 🏛️ أولاً: التصميم الهيكلي وتوزيع الأقسام

تمت إعادة هيكلة التطبيق من تصميم مبني على نوافذ منبثقة (Modals) تقليدية إلى ما يشبه **المتحف الرقمي** الذي يتضمن "قاعات" مستقلة تمنح القارئ تجربة انتقال غامرة.

### الواجهة الرئيسية (قاعة الدخول)
تم تصميم واجهة استقبال (Home Page) مركزية توفر إمكانية التنقل بين القاعات عبر خريطة مرئية، مما يخلق شعوراً بالدخول إلى فضاء روحي منظم:
- **القصة (اللون الذهبي):** متابعة القراءة واستعراض السرد التاريخي.
- **مسير القافلة (اللون الذهبي/البرتقالي):** رحلة الخروج من المدينة حتى كربلاء.
- **الشهداء (اللون الأحمر الغامق):** معجم الشهداء.
- **ديوان الطف (اللون الأزرق الداكن):** الأشعار والأقوال.
- **الزيارات (اللون الأخضر الزمردي):** نصوص الزيارات.
- **حول أثر (اللون الرمادي):** التعريف بالمشروع.

### القاعات والميزات المساعدة (Halls)
- **مستعرض القصة (Story View):** القسم المركزي في التطبيق، يعرض فصول ملحمة كربلاء (17 فصلاً) بتنسيق نصوص مريح للعين، ويدعم التنقل السلس بين الفصول مع تأثيرات حركية أنيقة، بالإضافة إلى تنسيق خاص للأشعار والخطب داخل النص التاريخي.
- **الخط الزمني للمسير (Timeline Hall):** واجهة بصرية تتتبع مسار ركب الإمام الحسين (ع) جغرافياً وزمنياً، بدءاً من الخروج من المدينة المنورة وصولاً إلى أرض كربلاء. صُممت على شكل محطات، يظهر فيها الحدث، التاريخ، اقتباس من الرواية، وإمكانية الانتقال للفصل المقابل في القصة.
- **معجم الشهداء (Martyrs Hall):** سجل وقاعدة بيانات شاملة لشهداء الطف من أهل البيت (ع) والأصحاب. صمم بأسلوب "بطاقات أثرية" ذات خلفية تحاكي الورق القديم، تتمدد عند الضغط عليها لعرض السيرة والمقتل بشكل منسق. تم تحديث المعجم بأسماء جميع الشهداء المذكورين في المجلد.
- **ديوان الطف (Diwan Hall):** مكتبة تضم الأراجيز، الأشعار، الخطب، والكلمات الخالدة، مصممة على شكل "رفوف مكتبة". تم استخراج النصوص والأقوال والحكم والأدعية وأبيات الشعر وإضافتها **كاملة دون أي اختصار** مع تزويد هذا القسم بميزة **(إظهار النص كاملاً / طي النص)** للنصوص الطويلة.
- **زيارات الطف (Ziyarat Hall):** قسم مخصص للزيارات المأثورة، مُصمم بخلفية سوداء، ومزود بميزة مخصصة تدعى **(وضع القراءة)**، التي تقوم بإخفاء أزرار الواجهة وإبقاء النص بحجم كبير مريح، ليُحاكي تجربة قراءة كتاب حقيقي.

## 🤖 ثانياً: كيف يعمل بوت التطوير الذكي (AI Coding Agent)؟

يعتمد البوت على محرك ذكاء اصطناعي تفاعلي متقدم مدمج داخل بيئة تطوير السحاب (Google AI Studio Build) يعمل بنظام **العميل الوكيل الموجه بالمهام (Goal-Oriented Agentic System)**، وتتلخص آلية عمله في الخطوات التالية:

1. **فهم نية المستخدم وسياقه (Intent Detection)**: يقوم البوت بتحليل طلباتك بدقة، والتفريق بين الاستفسار المعرفي (تقديم الشروحات) والطلب الإجرائي (كتابة أو تعديل الكود)، مع الالتزام التام باللغة العربية والهوية البصرية الإسلامية للمشروع.
2. **التحليل التاريخي الصارم (Historical Data Parsing)**: يعتمد البوت عند تلقيه نصوصاً ومخطوطات تاريخية جديدة على تحليلها بشكل دقيق وتفكيكها منهجياً، مستخرجاً الأحداث والشخصيات والتواريخ وأهم الخطب والحوارات بنصها الأصلي.
3. **فحص الملفات وقراءتها (Read-Before-Write)**: لا يفترض البوت وجود كود معين مسبقاً، بل يستعمل أدوات الفحص المخصصة مثل `view_file` و`list_dir` لقراءة وتحديث السياق البرمجي الفعلي قبل اتخاذ أي خطوة.
4. **مراعاة قيود المنصة والأمان**:
   - العمل على المنفذ الافتراضي الموحد للشبكة `3000`.
   - استخدام مكاتب قياسية معتمدة مسبقاً في البيئة (مثل React 18، Tailwind CSS، Lucide React، Motion للأنيميشن).
5. **تعديل الكود الجراحي الدقيق (Surgical Edits)**: يُجري البوت تعديلاته بدقة عالية عبر أدوات لضمان عدم تلف الكود أو حدوث تعارضات، مع مراعاة المؤثرات الحركية (Fade, Blur, Parallax خفيف) والانتقالات البطيئة لإضفاء إحساس الفخامة.
6. **الاختبار والتثبت الذاتي (Self-Verification)**: بعد كل تعديل، يستدعي البوت تلقائياً أدوات الفحص لبناء التطبيق للتأكد من نجاح عملية التشغيل قبل تسليمه لك.

## 🛠️ رابعاً: تشغيل وبناء وتجربة التطبيق محلياً

لبدء تشغيل وتجربة التطبيق محلياً في بيئتك الخاصة، اتبع الخطوات القياسية التالية:

1. **تثبيت الحزم والتبعيات البرمجية**:
   ```bash
   npm install
   ```

2. **تشغيل سيرفر التطوير المحلي (Dev Server)**:
   ```bash
   npm run dev
   ```

3. **عمل فحص وتحقق من سلامة الأخطاء (Lint & Type-Check)**:
   ```bash
   npm run lint
   ```

4. **بناء النسخة الإنتاجية المستقرة للتوزيع ونشر الـ PWA**:
   ```bash
   npm run build
   ```
