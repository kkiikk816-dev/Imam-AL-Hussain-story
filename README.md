# أثَر | ملحمة كربلاء 🏴

تطبيق ويب تفاعلي شامل يهدف إلى توثيق وسرد واقعة الطف الخالدة (ملحمة كربلاء) استناداً إلى أمهات الكتب والمصادر التاريخية المعتبرة (مثل كتاب بحار الأنوار). يقدم التطبيق تجربة مستخدم روحانية وتاريخية فريدة تجمع بين أصالة المحتوى وحداثة التصميم التفاعلي.

## 📂 هيكلية المشروع والتنظيم (Project Structure)

يعتمد التطبيق على بنية برمجية منظمة وفق إطار عمل **React** (مع **Vite**) و **TypeScript**. تنقسم الملفات إلى مكونات (Components) لعرض الواجهات، وبيانات (Data) لتخزين المحتوى التاريخي.

### 1. الملفات الجذرية (Root Files)
- `src/main.tsx`: نقطة الإدخال (Entry Point) الرئيسية لتطبيق React.
- `src/App.tsx`: المكون الجذري للتطبيق. يتولى إدارة الحالة الشاملة للتطبيق (State Management) مثل التبديل بين الأروقة (الواجهات)، تتبع الفصل المقروء وموضع التمرير، إدارة حجم الخط، وتغيير السمة (النهارية والليلية). كما يضم مكونات التنقل الجانبي والسفلي.
- `src/index.css`: ملف التنسيق الرئيسي، حيث يتم استدعاء وتهيئة `Tailwind CSS`.
- `src/types.ts`: يحتوي على تعريفات الواجهات `TypeScript` (Interfaces) المستخدمة لضمان أمان الأنواع (Type Safety) لبيانات الفصول والأنواع المشتركة.

### 2. مكونات الواجهة الأساسية (Components - `src/components/`)
يحتوي هذا المجلد على الواجهات التي تمثل "أروقة" أو شاشات التطبيق الرئيسية:
- **`HomeView.tsx`**: واجهة البداية، تعرض العنوان الرئيسي واقتباساً عشوائياً، وتحتوي على أزرار للتنقل بين كافة الأروقة (البحث، الفصول، الشهداء، الخرائط، الإعدادات).
- **`StoryView.tsx`**: مكون مخصص لعرض محتوى الفصول (السرد التاريخي). يدعم تنسيقات متقدمة مثل تمييز العناوين، عرض الأبيات الشعرية بطريقة كلاسيكية متقابلة، تحديد النص ومشاركته أو نسخه (عبر نافذة عائمة)، وإدارة التنسيق حسب السمة.
- **`MartyrsHall.tsx`**: (معجم الشهداء) يعرض قائمة مفصلة بشهداء واقعة الطف مع إمكانية التصفية حسب الفئة (أهل البيت، الأنصار، الخ) والبحث. يعرض تفاصيل الشهيد كاسمه، طريقة استشهاده، موعدها، واسم قاتله.
- **`DiwanHall.tsx`**: (ديوان الطف) واجهة مخصصة لعرض النصوص، الأشعار، الأراجيز، والقصص التاريخية المتعلقة بالمعركة.
- **`ZiyaratHall.tsx`**: (رواق الزيارات) يعرض النصوص المأثورة لزيارات أهل البيت بأجواء روحانية، مع تصميم يحاكي القراءة من الصحف المكتوبة.
- **`TimelineHall.tsx`**: (فهرس الفصول) واجهة تعرض تسلسل فصول الملحمة (من الفصل 1 إلى 17) ليتسنى للمستخدم الانتقال السريع لأي محطة.
- **`JourneyMapView.tsx`**: (خريطة المسير) واجهة تفاعلية تصوّر رحلة الإمام الحسين (ع) جغرافياً والمحطات التاريخية التي مر بها.
- **`SearchView.tsx`**: محرك بحث شامل داخل التطبيق للبحث عن الكلمات أو النصوص في الفصول والشهداء لتسهيل الوصول للمعلومة.
- **`SettingsView.tsx`**: واجهة الإعدادات لتمكين المستخدم من التحكم بحجم خط القراءة والتبديل بين الوضع الليلي (Dark) والنهاري (Light).
- **`AmbientAudio.tsx`**: مكون مخصص للتحكم في الصوتيات الخلفية أو المؤثرات الروحانية المرفقة.

### 3. قاعدة البيانات المحلية (Data Directory - `src/data/`)
هذا المجلد يحتوي على المحتوى التاريخي للتطبيق، وهو معزول تماماً عن المنطق البرمجي لتسهيل عملية التحديث:
- **`chapters/chapter[1-17].ts`**: سلسلة من 17 ملفاً برمجياً تمثل فصول القصة مرتبة زمنياً. كل ملف يصدر كائناً يضم `title` (العنوان)، `description` (وصف موجز)، و `content` (نص الفصل كاملاً مدعوماً بتنسيقات خاصة للنصوص العريضة والأشعار).
- **`martyrs.ts`**: مصفوفة من الكائنات (Objects) تضم بيانات دقيقة عن أصحاب الحسين (ع) وأهل بيته الذين استشهدوا. تشمل البيانات: الاسم، اللقب، الفئة، طريقة الموت، التوقيت، المكان، القاتل، ووصف الموقف.
- **`poems.ts`**: مصفوفة نصوص توثق المراثي، الأراجيز البطولية في المعركة، والمواقف الأدبية.
- **`quotes.ts`**: مقولات مأثورة وخطب للإمام الحسين (ع) تُعرض بشكل عشوائي في الواجهة الرئيسية `HomeView`.
- **`types.ts`**: ملف يجمع ويعيد تصدير الواجهات المعرفة مسبقاً.
- **`update_data.cjs` / `update_ziyarat.js`**: نصوص برمجية (Scripts - Node.js) تُستخدم من قبل المطور لحقن وتحديث البيانات بشكل آلي في ملفات `ts`.

## 🔄 آلية عمل التطبيق وإدارة الحالة (Workflow & State)

- **إدارة التوجيه الداخلي (Navigation State):** لا يعتمد التطبيق على مكتبات التوجيه المعتادة (مثل React Router)، بل يعتمد على حالة داخلية (State) في `App.tsx` تحت مسمى `currentView`. يتم التبديل بين المكونات بناءً على هذه الحالة باستخدام مكون `AnimatePresence` من مكتبة `Framer Motion` لإضافة حركات انتقال (Transitions) سلسة بين الشاشات.
- **التخزين المحلي (Local Storage):** لضمان استمرارية تجربة المستخدم، يقوم التطبيق بحفظ:
  - رقم الفصل الحالي (`karbala_current_chapter`).
  - موضع التمرير لكل فصل (`karbala_chapter_progress`) ليتمكن المستخدم من استكمال القراءة من حيث توقف.
  - تفضيلات الواجهة مثل حجم الخط (`karbala_font_size`) والسمة (`karbala_theme`).
- **واجهة القراءة (StoryView):** تقوم دالة `formatContent` بمعالجة نص الفصل وتقسيمه إلى فقرات، وتمييز الكلمات المحاطة بعلامات `**` لتبدو بخط عريض، وتنسيق الأبيات الشعرية المفصولة بعلامات `***` لتظهر كشطر وعجز متقابلين. كما يستمع المكون لتحديد النص لإظهار أدوات النسخ والمشاركة المدمجة.

## 🛠️ التقنيات المستخدمة (Technologies)

- **React (v18):** لبناء واجهات مستخدم تفاعلية بنظام المكونات.
- **Vite:** كأداة بناء (Build Tool) وخادم تطوير سريع يضمن سرعة التحديث أثناء البرمجة.
- **TypeScript:** لكتابة كود آمن (Type-Safe) ومنع الأخطاء في وقت التصميم.
- **Tailwind CSS:** لإدارة تنسيقات الواجهة بشكل مرن وسريع، وتوفير الدعم المدمج للسمات الليلية والنهارية.
- **Motion (Framer Motion):** لإضافة انتقالات حركية ناعمة تعزز التفاعل (مثل التبديل بين الأروقة أو فتح القائمة الجانبية المنسدلة).
- **Lucide React:** حزمة توفر أيقونات SVG أنيقة ومتجاوبة للواجهة.

## 🚀 طريقة التشغيل للمطورين (How to Run)

1. تثبيت الحزم البرمجية:
   ```bash
   npm install
   ```
2. تشغيل خادم التطوير:
   ```bash
   npm run dev
   ```
3. بناء التطبيق للإنتاج:
   ```bash
   npm run build
   ```
