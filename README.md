# أثَر | ملحمة كربلاء 🌟
> **رؤية رقمية معاصرة لنشر سيرة الإمام الحسين (ع) ورجالات الطف البررة**

منصة **أثَر** الرقمية هي تطبيق تفاعلي ويب متكامل ومصقول يوثق الأحداث الكاملة لـ **ملحمة كربلاء** التاريخية وسيرة سيد الشهداء الإمام الحسين بن علي (ع) وأصحابه البررة، بالاعتماد على النصوص التاريخية من كتاب **بحار الأنوار** للعلامة المجلسي. يهدف التطبيق إلى تقديم هذه الملحمة المباركة بطريقة معاصرة تدمج بين جماليات التصميم والمزايا التفاعلية الرقمية لتسهيل القراءة والبحث والدراسة.

---

## 🏛️ أولاً: التصميم الهيكلي وتوزيع الأقسام

تم تصميم منصة "أثَر" لتوفير تجربة مستخدم سلسة وعميقة، حيث تم توزيع المحتوى بين أقسام رئيسية تقود السرد التاريخي، وأقسام جانبية (Modals) تثري المعرفة وتقدم تفاصيل دقيقة:

### الأقسام الرئيسية (Main Sections)
- **مستعرض القصة (Story View):** القسم المركزي في التطبيق، يعرض فصول ملحمة كربلاء (17 فصلاً) بتنسيق نصوص مريح للعين، ويدعم التنقل السلس بين الفصول مع تأثيرات حركية أنيقة، بالإضافة إلى تنسيق خاص للأشعار والخطب داخل النص التاريخي.
- **الخط الزمني للمسير (Journey Timeline):** واجهة بصرية تتتبع مسار ركب الإمام الحسين (ع) جغرافياً وزمنياً، بدءاً من الخروج من المدينة المنورة وصولاً إلى أرض كربلاء، لربط الأحداث بالمكان والزمان.

### الأقسام الجانبية والميزات المساعدة (Side Sections & Modals)
تم توفير المحتوى الإضافي عبر نوافذ منبثقة (Modals) لتجنب تشتيت القارئ أثناء تصفح الرواية الأساسية:
- **معجم الشهداء (Martyrs Glossary):** سجل وقاعدة بيانات شاملة لشهداء الطف من أهل البيت (ع) والأصحاب. يتميز ببطاقات تفاعلية تعرض اسم الشهيد، لقبه، الفئة، تفاصيل استشهاده، والقاتل. تم فحص النصوص وتحديث المعجم بأسماء جميع الشهداء المذكورين في المجلد (مثل أنس بن حارث، عمرو بن جنادة، عبد الله بن مسلم وغيرهم) بنفس طريقة العرض المتفق عليها.
- **ديوان الطف (Diwan):** مكتبة تضم الأراجيز، الأشعار، الخطب، والكلمات الخالدة. **ميزة هامة:** تم استخراج النصوص والأقوال والحكم والأدعية وأبيات الشعر من المصادر وإضافتها **كاملة دون أي اختصار**. ولضمان راحة القارئ، تم تزويد هذا القسم بميزة **(إظهار النص كاملاً / طي النص)** للنصوص الطويلة.
- **زيارات الطف (Ziyarat):** قسم مخصص للزيارات المأثورة للإمام الحسين (ع) والشهداء، مصممة بخطوط عربية أصيلة لتجربة قراءة روحانية.
- **حول المنصة (About):** نافذة تعريفية توضح رؤية المنصة الرقمية، مزودة بشعار "أثَر" الملحمي.

## 🤖 ثانياً: كيف يعمل بوت التطوير الذكي (AI Coding Agent)؟

يعتمد البوت على محرك ذكاء اصطناعي تفاعلي متقدم مدمج داخل بيئة تطوير السحاب (Google AI Studio Build) يعمل بنظام **العميل الوكيل الموجه بالمهام (Goal-Oriented Agentic System)**، وتتلخص آلية عمله في الخطوات التالية:

1. **فهم نية المستخدم وسياقه (Intent Detection)**: يقوم البوت بتحليل طلباتك بدقة، والتفريق بين الاستفسار المعرفي (تقديم الشروحات) والطلب الإجرائي (كتابة أو تعديل الكود)، مع الالتزام التام باللغة العربية والهوية البصرية الإسلامية للمشروع.
2. **التحليل التاريخي الصارم (Historical Data Parsing)**: يعتمد البوت عند تلقيه نصوصاً ومخطوطات تاريخية جديدة على تحليلها بشكل دقيق وتفكيكها منهجياً، مستخرجاً الأحداث والشخصيات والتواريخ وأهم الخطب والحوارات بنصها الأصلي (بدون أي تخيل أو تحريف - Zero Hallucination) وتحويلها إلى مصفوفات وهياكل JSON. **تم استخراج الأقوال، والحكم، والأشعار كاملةً كما طُلب، مع إضافة الشهداء الجدد وتفاصيلهم.**
3. **فحص الملفات وقراءتها (Read-Before-Write)**: لا يفترض البوت وجود كود معين مسبقاً، بل يستعمل أدوات الفحص المخصصة مثل `view_file` و`list_dir` لقراءة وتحديث السياق البرمجي الفعلي قبل اتخاذ أي خطوة.
4. **مراعاة قيود المنصة والأمان**:
   - العمل على المنفذ الافتراضي الموحد للشبكة `3000`.
   - عزل مفاتيح الواجهات البرمجية (API Keys) في الخادم (Backend) وعدم تسريبها للواجهة الأمامية لمنع الاختراقات.
   - استخدام مكاتب قياسية معتمدة مسبقاً في البيئة (مثل React 18، Tailwind CSS، Lucide React، Motion للأنيميشن).
5. **تعديل الكود الجراحي الدقيق (Surgical Edits)**: يُجري البوت تعديلاته بدقة عالية عبر أدوات مثل `edit_file` و`multi_edit_file` لضمان عدم تلف الكود أو حدوث تعارضات.
6. **الاختبار والتثبت الذاتي (Self-Verification)**: بعد كل تعديل، يستدعي البوت تلقائياً أدوات الفحص لبناء التطبيق للتأكد من نجاح عملية التشغيل قبل تسليمه لك.

## 📑 رابعاً: توثيق ملفات المشروع البرمجية والتخزينية بالتفصيل

نستعرض هنا تفاصيل كافة الملفات في المشروع، وطبيعة عملها:

### 1. الملفات البرمجية الأساسية ومكونات العرض (Components)
| اسم الملف | الحالة | طريقة العمل والتفصيل التقني |
| :--- | :---: | :--- |
| **واجهة التطبيق الرئيسية (`App.tsx`)** | مكتمل ومعدل | قلب التطبيق؛ يتحكم بالتنقل بين القصة الرئيسية والأقسام الجانبية (Modals)، ويحتوي على خيارات العرض (الثيم المظلم/المضيء، حجم الخط) وحفظها. |
| **مستعرض القصة (`StoryView.tsx`)** | مكتمل ومعدل | مكون العرض الرئيسي للفصول التاريخية، مزود بتنسيق تلقائي يميز الأشعار والخطوط العريضة ويضفي طابعاً روحانياً وأنيقاً. |

| **نافذة التعريف (`AboutModal.tsx`)** | مكتمل | نافذة تعرض رؤية "أثَر" مع شعارها الملحمي. |

### 2. ملفات تخزين البيانات والمحتوى (Data Storage)
تحتوي قاعدة البيانات المحلية للتطبيق على رواية تفصيلية مقسمة لـ 17 فصلاً مقتبساً من بحار الأنوار، بالإضافة إلى قاعدة معلومات تفصيلية عن عمالقة ملحمة الطف.

* **نصوص الرواية (`/src/data/chapters/`):** من `chapter1` إلى `chapter17`، تمثل السرد التاريخي الكامل بدقة.
* **قاعدة بيانات الشهداء (`/src/data/martyrs.ts`):** مصفوفة كائنات تضم تفاصيل موثقة لكل شهيد. تم تحديث القاعدة لتشمل الشهداء المذكورين في النصوص مثل (عمرو بن جنادة، يحيى بن سليم، قرة بن أبي قرة، عبد الله بن مسلم، وغيرهم) مع الحفاظ على طريقة العرض المعتمدة.

## 🛠️ خامساً: تشغيل وبناء وتجربة التطبيق محلياً

لبدء تشغيل وتجربة التطبيق محلياً في بيئتك الخاصة، اتبع الخطوات القياسية التالية:

1. **تثبيت الحزم والتبعيات البرمجية**:
   ```bash
   npm install
   ```

2. **تشغيل سيرفر التطوير المحلي (Dev Server)**:
   ```bash
   npm run dev
   ```
   *سيقوم السيرفر بالعمل على المنفذ `3000` افتراضياً لفتح المعاينة التفاعلية.*

3. **عمل فحص وتحقق من سلامة الأخطاء (Lint & Type-Check)**:
   ```bash
   npm run lint
   ```

4. **بناء النسخة الإنتاجية المستقرة للتوزيع ونشر الـ PWA**:
   ```bash
   npm run build
   ```
   *ينتج هذا الأمر مجلد `dist/` جاهزاً للنشر المباشر على خوادم الاستضافة السحابية.*
