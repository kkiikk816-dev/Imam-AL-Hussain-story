# أثَر | ملحمة كربلاء 🌟
> **رؤية رقمية معاصرة لنشر سيرة الإمام الحسين (ع) ورجالات الطف البررة**

منصة **أثَر** الرقمية هي تطبيق تفاعلي ويب متكامل ومصقول يوثق الأحداث الكاملة لـ **ملحمة كربلاء** التاريخية وسيرة سيد الشهداء الإمام الحسين بن علي (ع) وأصحابه البررة، بالاعتماد على النصوص التاريخية من كتاب **بحار الأنوار** للعلامة المجلسي. يهدف التطبيق إلى تقديم هذه الملحمة المباركة بطريقة معاصرة تدمج بين جماليات التصميم والمزايا التفاعلية الرقمية لتسهيل القراءة والبحث والدراسة.

## 🤖 أولاً: كيف يعمل بوت التطوير الذكي (AI Coding Agent)؟

يعتمد البوت على محرك ذكاء اصطناعي تفاعلي متقدم مدمج داخل بيئة تطوير السحاب (Google AI Studio Build) يعمل بنظام **العميل الوكيل الموجه بالمهام (Goal-Oriented Agentic System)**، وتتلخص آلية عمله في الخطوات التالية:

1. **فهم نية المستخدم وسياقه (Intent Detection)**: يقوم البوت بتحليل طلباتك بدقة، والتفريق بين الاستفسار المعرفي (تقديم الشروحات) والطلب الإجرائي (كتابة أو تعديل الكود)، مع الالتزام التام باللغة العربية والهوية البصرية الإسلامية للمشروع.
2. **التحليل التاريخي الصارم (Historical Data Parsing)**: يعتمد البوت عند تلقيه نصوصاً ومخطوطات تاريخية جديدة على تحليلها بشكل دقيق وتفكيكها منهجياً، مستخرجاً الأحداث والشخصيات والتواريخ وأهم الخطب والحوارات بنصها الأصلي (بدون أي تخيل أو تحريف - Zero Hallucination) وتحويلها إلى مصفوفات وهياكل JSON ومقاطع Markdown موثقة بالكامل لتعرض بتسلسل روائي جذاب.
3. **فحص الملفات وقراءتها (Read-Before-Write)**: لا يفترض البوت وجود كود معين مسبقاً، بل يستعمل أدوات الفحص المخصصة مثل `view_file` و`list_dir` لقراءة وتحديث السياق البرمجي الفعلي قبل اتخاذ أي خطوة.
3. **مراعاة قيود المنصة والأمان**:
   - العمل على المنفذ الافتراضي الموحد للشبكة `3000`.
   - عزل مفاتيح الواجهات البرمجية (API Keys) في الخادم (Backend) وعدم تسريبها للواجهة الأمامية لمنع الاختراقات.
   - استخدام مكاتب قياسية معتمدة مسبقاً في البيئة (مثل React 18، Tailwind CSS، Lucide React، Motion للأنيميشن).
4. **تعديل الكود الجراحي الدقيق (Surgical Edits)**: يُجري البوت تعديلاته بدقة عالية عبر أدوات مثل `edit_file` و`multi_edit_file` لضمان عدم تلف الكود أو حدوث تعارضات.
5. **الاختبار والتثبت الذاتي (Self-Verification)**: بعد كل تعديل، يستدعي البوت تلقائياً أدوات الفحص مثل `lint_applet` للتأكد من خلو المشروع من أخطاء السنتاكس (Syntax) أو التايب سكريبت، وأداة `compile_applet` لبناء التطبيق للتأكد من نجاح عملية التشغيل قبل تسليمه لك.

  2. **الأخوان عبد الله وعبد الرحمن الغفاريان**: الأخوان الباكيان شوقاً للشهادة بين يدي الإمام.
  3. **يزيد بن زياد (أبو الشعثاء الكندي)**: الرامي البطل الذي سدد مئة سهم ودعا له الإمام.
  4. **الحجاج بن مسروق الجعفي**: مؤذن الإمام الحسين (ع) الفدائي.
  5. **عبد الله بن يقطر**: سفير ومبعوث الحسين المستشهد قبل كربلاء.

## 🛠️ رابعاً: تشغيل وبناء وتجربة التطبيق محلياً

لبدء تشغيل وتجربة التطبيق محلياً في بيئتك الخاصة، اتبع الخطوات القياسية التالية:

1. **تثبيت الحزم والتبعيات البرمجية**:
   ```bash
   npm install
   ```

2. **تشغيل سيرفر التطوير المحلي (Dev Server)**:
   ```bash
   npm run dev
   ```
   *سيقوم السيرفر بالعمل على المنفذ `3000` افتراضياً لفتح المعاينة التفاعلية.*

3. **عمل فحص وتحقق من سلامة الأخطاء (Lint & Type-Check)**:
   ```bash
   npm run lint
   ```

4. **بناء النسخة الإنتاجية المستقرة للتوزيع ونشر الـ PWA**:
   ```bash
   npm run build
   ```
   *ينتج هذا الأمر مجلد `dist/` جاهزاً للنشر المباشر على خوادم الاستضافة السحابية.*
